# Ticket 05 - 今日概览、趋势页、快捷入口与整体 UI 集成

## 目标

把宝宝档案、喂养、大小便和黄疸模块聚合成第一版可用体验：今日页一眼显示核心状态，记录页通过快捷入口进入三类表单，趋势页展示近几天变化，并保证整体视觉符合低噪、纸张感、夜间友好的设计方向。

## 开发范围

- 实现今日概览聚合：
  - `overviewService`
  - 今日总奶量
  - 今日喂养次数
  - 今日奶量状态
  - 今日尿尿次数
  - 今日便便次数
  - 最近一次黄疸平均值
  - 黄疸风险标志
  - 距离上次黄疸记录的时间
  - 黄疸较上次是否上升
- 实现趋势聚合和趋势页展示：
  - `trendService`
  - 喂养趋势
  - 大小便趋势
  - 黄疸趋势
  - 近 7 / 14 / 30 天通用范围
  - 黄疸近 3 / 7 / 14 天 / 全部范围
- 实现首页快捷操作：
  - 记喂奶
  - 记大小便
  - 记黄疸
- 实现核心复用组件：
  - `risk-badge`
  - `overview-metric`
  - `record-sheet`
  - `trend-chart`
  - `record-card`
  - `date-filter`
  - `empty-state`
- 完成底部 Tab 和页面集成：
  - 今日
  - 记录
  - 趋势
  - 宝宝
- 按 UI 设计说明落地视觉：
  - 极简、低噪、纸张感
  - 今日页用超大奶量数字承载主状态
  - 风险提示克制
  - 卡片和按钮圆角约 8px
  - 主色 `#20211F`
  - 背景 `#F7F7F2`
  - 页面主体 `#FFFEF9`

## 不包含

- 新增登录、云同步、多宝宝、多成员协作。
- 数据导出。
- 喂奶提醒或黄疸复查提醒。
- 修改各业务 rule 的核心算法；本 Ticket 只聚合和展示各 service 输出。

## 技术要求

- 今日页不得直接读取 repository，必须调用 `overviewService`。
- 趋势页不得直接拼装业务统计，必须调用 `trendService` 或各模块 service 暴露的数据接口。
- 组件只处理 UI 表现，不直接访问本地存储。
- 空数据状态必须可用：
  - 未创建宝宝档案
  - 无今日喂养记录
  - 无大小便记录
  - 无黄疸记录
  - 无趋势数据
- 所有健康相关提示统一遵守医疗提示原则，不输出绝对化医疗结论。

## 验收标准

- 底部 Tab 可以在今日、记录、趋势、宝宝之间切换。
- 今日页有一个明显的今日总奶量大数字区域。
- 今日页展示今日喂养次数、奶量状态、尿尿次数、便便次数。
- 今日页展示最近一次黄疸平均值、风险标志、距离上次记录时间。
- 当至少有两条黄疸记录时，今日页展示最近黄疸较上次上升、下降或持平。
- 今日页缺少宝宝体重或出生日期时，奶量状态展示“无法判断”。
- 今日页缺少黄疸判断所需字段时，黄疸风险展示“无法判断”。
- 今日页没有黄疸记录时，最近黄疸区域展示空状态，不报错。
- 点击“记喂奶”可以进入喂养新增流程。
- 点击“记大小便”可以进入大小便新增流程。
- 点击“记黄疸”可以进入黄疸新增流程。
- 记录页可以在喂养、大小便、黄疸三个分段之间切换。
- 记录页历史卡片使用倒序展示，并支持接入各模块的编辑、删除、日期筛选能力。
- 趋势页可以切换喂养、大小便、黄疸趋势。
- 喂养趋势支持近 7 / 14 / 30 天范围，并展示每日总奶量、喂养次数和建议奶量区间。
- 大小便趋势支持近 7 / 14 / 30 天范围，并展示尿尿次数、便便次数、量分布和特殊颜色时间线。
- 黄疸趋势支持近 3 / 7 / 14 天和全部范围，并展示平均值、三个测量值、风险颜色和升降变化。
- `risk-badge` 可以展示正常、低危、中危、高危、无法判断、不适用。
- 所有健康提示文案包含家庭记录和观察参考含义，不替代医生判断。
- 页面视觉符合设计说明：低噪、纸张感、温和但不幼稚，主状态突出，辅助信息层级更轻。
- 在无登录、无云同步、无提醒能力的情况下，MVP 仍能完整完成本地记录、查看、编辑、删除、概览和趋势查看。

## 独立测试方式

- 使用 mock services 或本地 seed 数据测试 `overviewService`：
  - 无宝宝档案
  - 有宝宝档案但无记录
  - 有今日喂养 / 大小便 / 黄疸记录
  - 黄疸上升、下降、持平
- 使用 mock trend data 测试趋势页，不依赖真实业务表单完成。
- 在微信开发者工具中完成端到端手测：
  - 创建宝宝档案
  - 新增喂养记录
  - 新增大小便记录
  - 新增黄疸记录
  - 回到今日页检查聚合数据
  - 进入趋势页检查范围切换
- 该 Ticket 可以通过 mock 数据独立验收 UI 和聚合逻辑；接入真实模块后再进行一次轻量回归。
