# Ticket 03 - 大小便记录、颜色提示、历史列表与排泄趋势数据

## 目标

完成大小便模块的完整纵向切片：用户可以记录尿尿、便便或两者同时发生，系统可以统计每日尿便次数、量分布和需关注颜色，并提供趋势页可消费的数据。

## 开发范围

- 实现大小便数据模型、repository、service 和 rule：
  - `diaperRepository`
  - `diaperService`
  - `diaperRule`
- 支持新增大小便记录字段：
  - 类型：尿尿 / 便便 / 尿尿 + 便便
  - 记录时间，默认当前时间
  - 尿量：小 / 中 / 大
  - 粪量：小 / 中 / 大
  - 颜色：黄色 / 绿色 / 棕色 / 黑色 / 红色 / 灰白色或白色 / 其他
  - 备注
- 支持字段显示规则：
  - 选择尿尿时，显示尿量和颜色
  - 选择便便时，显示粪量和颜色
  - 选择尿尿 + 便便时，显示尿量、粪量和颜色
- 支持大小便历史：
  - 默认按记录时间倒序
  - 按日期筛选
  - 编辑记录
  - 删除记录，采用软删除
  - 查看备注
- 支持每日统计：
  - 尿尿总次数
  - 便便总次数
  - 尿量分布
  - 粪量分布
  - 特殊颜色次数
- 支持大小便趋势数据输出：
  - 近 7 天
  - 近 14 天
  - 近 30 天
  - 每日尿尿次数
  - 每日便便次数
  - 尿量分布
  - 粪量分布
  - 特殊颜色记录时间线

## 不包含

- 今日页完整聚合展示。
- 趋势页最终图表 UI 绘制。
- 喂养和黄疸模块。
- 医疗诊断建议。

## 技术要求

- 特殊颜色判断必须放在 `diaperRule`，页面只展示 rule 输出结果。
- 以下颜色标记为需关注：
  - 黑色
  - 红色
  - 灰白色或白色
- 健康提示文案必须克制，使用“请结合宝宝状态观察，必要时咨询医生”一类表达。
- 选择不同记录类型时，service 必须校验对应必填字段：
  - 尿尿：尿量和颜色必填
  - 便便：粪量和颜色必填
  - 尿尿 + 便便：尿量、粪量和颜色必填

## 验收标准

- 可以在记录页选择“大小便”分段并新增尿尿记录。
- 可以在记录页选择“大小便”分段并新增便便记录。
- 可以在记录页选择“大小便”分段并新增尿尿 + 便便记录。
- 选择尿尿时，表单只要求尿量和颜色，不要求粪量。
- 选择便便时，表单只要求粪量和颜色，不要求尿量。
- 选择尿尿 + 便便时，表单同时要求尿量、粪量和颜色。
- 新增记录时，不手动选择时间则默认使用当前时间。
- 保存成功后，大小便历史列表立即出现该记录，且按时间倒序排列。
- 可以按日期筛选大小便历史记录。
- 可以编辑一条大小便记录的类型、量、颜色、时间和备注。
- 可以删除一条大小便记录，删除后默认列表和统计不再包含该记录。
- 同一天新增尿尿、便便、尿尿 + 便便记录后，尿尿总次数和便便总次数分别正确累计。
- 尿量分布能正确统计小 / 中 / 大。
- 粪量分布能正确统计小 / 中 / 大。
- 颜色为黑色、红色、灰白色或白色时，记录卡片和统计结果中出现需关注标识。
- 颜色为黄色、绿色、棕色或其他时，不出现需关注标识。
- 趋势数据接口在近 7 / 14 / 30 天范围内返回连续自然日数据，缺少记录的日期返回 0 值。
- 特殊颜色记录时间线包含记录时间、颜色、记录类型和提示级别。

## 独立测试方式

- 使用 mock repository 写入不同类型、不同颜色、不同日期的大小便记录，测试每日统计和趋势数据。
- 单独测试 `diaperRule`：
  - 黑色 / 红色 / 灰白色或白色返回需关注
  - 黄色 / 绿色 / 棕色 / 其他返回普通
- 在微信开发者工具中只进入记录页大小便分段，完成新增、编辑、删除、日期筛选流程。
- 该 Ticket 可在没有今日页和趋势图最终 UI 的情况下验收；趋势只验收数据结构和范围切换结果。
