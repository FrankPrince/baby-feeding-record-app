# Ticket 04 - 黄疸记录、平均值、风险标志、历史列表与黄疸趋势数据

## 目标

完成黄疸模块的完整纵向切片：用户可以记录 3 个 mg/dL 黄疸值，系统自动计算平均值，并结合宝宝信息输出克制的风险标志和趋势数据。黄疸模块在第一版中为高优先级，后续首页需要突出展示其最近状态和变化。

## 开发范围

- 实现黄疸数据模型、repository、service 和 rule：
  - `jaundiceRepository`
  - `jaundiceService`
  - `jaundiceRule`
- 支持新增黄疸记录字段：
  - 黄疸值 1，单位 mg/dL
  - 黄疸值 2，单位 mg/dL
  - 黄疸值 3，单位 mg/dL
  - 平均值，系统自动计算
  - 测量方式：经皮 / 血清 / 未知
  - 记录时间，默认当前时间
  - 自动风险标志
  - 备注
- 支持平均值计算：
  - 三个值都必填
  - 只支持 mg/dL
  - 平均值保留 1 位小数
- 支持风险标志：
  - 正常
  - 低危
  - 中危
  - 高危
  - 无法判断
  - 不适用
- 支持黄疸历史：
  - 默认按记录时间倒序
  - 按日期筛选
  - 编辑记录
  - 删除记录，采用软删除
  - 查看备注
- 支持黄疸趋势数据输出：
  - 近 3 天
  - 近 7 天
  - 近 14 天
  - 全部
  - 平均值折线数据
  - 三个测量值散点数据
  - 风险标志颜色
  - 风险区间背景数据
  - 与上一条记录相比的升降变化

## 不包含

- 今日页完整聚合展示。
- 趋势页最终图表 UI 绘制。
- 真实医疗诊断或医生远程查看。
- 黄疸复查提醒。

## 技术要求

- 平均值计算和风险判断必须放在 `jaundiceRule`，页面不得直接计算风险等级。
- 风险判断输入必须包含：
  - 出生后小时数
  - 出生孕周
  - 黄疸平均值
  - 是否存在神经毒性风险因素
  - 测量方式
- 宝宝孕周小于 35 周时，返回“不适用”，并提示建议医生判断。
- 缺少出生时间、出生孕周或黄疸值时，返回“无法判断”。
- PRD 未给出精确黄疸阈值时，第一版实现必须把风险阈值表集中放在 rule 配置中，页面只消费风险结果和文案。
- 所有黄疸文案必须遵守医疗提示原则：
  - 不能使用确诊、必须治疗、不用担心等绝对化表述
  - 常规提示使用“本结果仅用于家庭记录和观察，不能替代医生判断”
  - 高危提示使用“当前黄疸值较高，建议尽快咨询医生”

## 验收标准

- 可以在记录页选择“黄疸”分段并新增黄疸记录。
- 黄疸值 1、2、3 均必填；任一为空时保存失败并提示。
- 黄疸值只接受有效数字；非数字、等于 0 或小于 0 时保存失败并提示。
- 输入 8.1、8.6、9.0 时，平均值自动保存为 8.6 mg/dL。
- 新增记录时，不手动选择时间则默认使用当前时间。
- 保存成功后，黄疸历史列表立即出现该记录，且按时间倒序排列。
- 可以按日期筛选黄疸历史记录。
- 可以编辑一条黄疸记录的 3 个数值、测量方式、时间和备注。
- 编辑数值后，平均值和风险标志重新计算。
- 可以删除一条黄疸记录，删除后默认列表、最近状态和趋势数据不再包含该记录。
- 宝宝孕周小于 35 周时，风险标志为“不适用”。
- 缺少宝宝出生时间、出生孕周或黄疸值时，风险标志为“无法判断”。
- 数据完整且命中不同阈值区间时，风险标志可分别返回正常、低危、中危、高危。
- 高危记录展示高关注文案：“当前黄疸值较高，建议尽快咨询医生。”
- 常规黄疸记录展示家庭观察参考文案，不输出诊断结论。
- 黄疸趋势数据支持近 3 / 7 / 14 天和全部范围。
- 趋势数据包含每条记录的平均值、3 个原始测量值、风险等级、风险颜色和较上一条的升降变化。

## 独立测试方式

- 使用 mock baby profile 和 mock repository 单独测试 `jaundiceRule`：
  - 平均值保留 1 位小数
  - 孕周小于 35 周返回不适用
  - 缺少关键字段返回无法判断
  - 阈值边界返回正常 / 低危 / 中危 / 高危
- 使用多条不同时间的黄疸记录测试近 3 / 7 / 14 天和全部趋势数据。
- 在微信开发者工具中只进入记录页黄疸分段，完成新增、编辑、删除、日期筛选流程。
- 该 Ticket 可在没有今日页和趋势图最终 UI 的情况下验收；趋势只验收数据结构和风险文案输出。
