# Ticket 02 - 喂养记录、奶量规则、历史列表与喂养趋势数据

## 目标

完成喂养模块的完整纵向切片：用户可以新增、查看、编辑、删除喂养记录；系统可以按自然日统计奶量和喂养次数，并基于宝宝体重和日龄 / 月龄输出奶量状态。

## 开发范围

- 实现喂养数据模型、repository、service 和 rule：
  - `feedingRepository`
  - `feedingService`
  - `feedingRule`
- 支持新增喂养记录字段：
  - 喂养类型：母乳 / 奶粉
  - 奶量 ml
  - 喂养时间，默认当前时间
  - 备注
- 支持奶量输入交互：
  - 快捷选择 30 / 60 / 90 / 120 ml
  - 手动输入 ml
  - 母乳也按 ml 记录
- 支持喂养历史：
  - 默认按喂养时间倒序
  - 按日期筛选
  - 编辑记录
  - 删除记录，采用软删除
  - 查看备注
- 支持每日喂养统计：
  - 总奶量
  - 总喂养次数
  - 母乳总量
  - 奶粉总量
  - 母乳次数
  - 奶粉次数
  - 奶量状态
- 支持喂养趋势数据输出：
  - 近 7 天
  - 近 14 天
  - 近 30 天
  - 每日总奶量
  - 每日喂养次数
  - 建议奶量上下限
  - 母乳 / 奶粉占比

## 不包含

- 今日页完整聚合展示。
- 趋势页最终图表 UI 绘制。
- 大小便和黄疸模块。
- 喂奶提醒、亲喂时长、左右侧母乳记录。

## 技术要求

- 奶量状态判断必须放在 `feedingRule`，页面不得直接写判断逻辑。
- 每日建议奶量区间通过集中配置表计算：
  - 输入：当前体重 kg、出生后日龄 / 月龄、当日总奶量 ml
  - 输出：建议下限、建议上限、偏低 / 达标 / 偏高 / 无法判断
- PRD 未给出精确医学区间时，第一版实现必须把 ml/kg/day 区间放在单独常量或配置中，并注明后续可按医生建议调整。
- 缺少宝宝体重或出生日期时，返回“无法判断”，不得给出伪精确结论。
- 所有健康相关文案必须包含家庭记录参考含义，不替代医生建议。

## 验收标准

- 可以在记录页选择“喂养”分段并新增母乳记录。
- 可以在记录页选择“喂养”分段并新增奶粉记录。
- 新增记录时，不手动选择时间则默认使用当前时间。
- 奶量可以通过 30 / 60 / 90 / 120 ml 快捷值填写。
- 奶量可以通过手动输入填写。
- 奶量为空、非数字、等于 0 或小于 0 时，保存失败并给出表单提示。
- 保存成功后，喂养历史列表立即出现该记录，且按时间倒序排列。
- 可以按日期筛选喂养历史记录。
- 可以编辑一条喂养记录的类型、奶量、时间和备注。
- 可以删除一条喂养记录，删除后默认列表和统计不再包含该记录。
- 同一天新增 2 条母乳和 1 条奶粉记录后，每日统计能正确返回总奶量、总次数、母乳总量、奶粉总量、母乳次数、奶粉次数。
- 当日总奶量低于建议下限时，奶量状态为“偏低”。
- 当日总奶量位于建议区间时，奶量状态为“达标”。
- 当日总奶量高于建议上限时，奶量状态为“偏高”。
- 缺少宝宝体重或出生日期时，奶量状态为“无法判断”。
- 趋势数据接口在近 7 / 14 / 30 天范围内返回连续自然日数据，缺少记录的日期返回 0 值而不是漏掉日期。
- 喂养趋势数据包含建议奶量上下限，可供趋势页绘制参考线。

## 独立测试方式

- 使用 mock baby profile 测试 `feedingRule` 的 4 类状态：偏低、达标、偏高、无法判断。
- 使用 mock repository 写入多天喂养记录，测试每日统计和 7 / 14 / 30 天趋势数据。
- 在微信开发者工具中只进入记录页喂养分段，完成新增、编辑、删除、日期筛选流程。
- 该 Ticket 可在没有今日页和趋势图最终 UI 的情况下验收；趋势只验收数据结构和范围切换结果。
